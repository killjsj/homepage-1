# 贡献指南

> 本页面由 AI 生成，并已经过人工修改和审核。

<!-- Reviewers: RhenCloud, wyf9 -->

本文档用于说明如何为 SiiWay 主页项目贡献内容与代码。

## 适用范围

- 文档内容修订（错别字、链接修复、内容补充）
- 页面结构与样式优化
- 构建与工具链配置改进

## 基本流程

1. Fork 仓库并创建分支。
2. 完成修改并通过本地检查。
3. 提交 Pull Request，说明修改目的与影响范围。
4. 等待维护者评审与合并。

## 提交建议

- 小步提交，避免一次 PR 混入多个不相关改动。
- 提交信息应清晰描述变更内容。
- 涉及规范变更时，请同步更新对应规范文档。

## 质量检查

提交前建议至少完成：

- 确保你的提交未违反 SiiWay 行为守则
- 拼写与链接自检
- 本地预览页面确认显示正常
- 对应语言版本内容一致性检查（如适用）
- 去除无用的 AI 提示文本
- 在提交更改前先询问 SiiWay Core 成员

## 标签符号使用规范

为保证文档风格一致，请遵循以下规则：

- 无序列表统一使用 `-`，避免同一文档中混用 `-` 与 `*`。
- 中文语境中的标签说明统一使用全角冒号（如“GitHub：”“Email：”）。
- 中英文混排时，中文括号优先使用全角（“（”与“）”）。
- 需要强调的平台名、术语可使用加粗，但尽量避免整段连续加粗。
- 链接统一写作 `[文本](链接)`，避免裸链接与混合样式。

如需在特定页面保留个性化写法，请在 PR 描述 / 提交消息 中说明理由。
